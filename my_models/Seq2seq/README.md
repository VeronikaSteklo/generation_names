# Seq2Seq (Encoder–Decoder with Attention)

Модуль реализует архитектуру Seq2Seq (Encoder–Decoder) с механизмом внимания (Attention) для генерации заголовков по
входному тексту.

Seq2Seq — классическая архитектура для задач условной генерации текста. Состоит из двух основных компонентов:

1. **Encoder** — кодирует входной текст в последовательность скрытых состояний, используя двунаправленную LSTM (
   Bidirectional LSTM). Финальные скрытое состояние и ячейка передаются декодеру как начальный контекст;
2. **Decoder** — генерирует заголовок токен за токеном, используя однонаправленную LSTM. На каждом шаге механизм
   внимания вычисляет веса по всем выходам энкодера, позволяя модели фокусироваться на наиболее релевантных участках
   исходного текста.

Механизм **Attention** решает проблему «бутылочного горлышка» классического Seq2Seq, где вся информация о тексте
сжималась в один фиксированный вектор. Теперь декодер на каждом шаге имеет доступ ко всем скрытым состояниям энкодера.

## Особенности архитектуры

- **Bidirectional LSTM Encoder**: входной текст обрабатывается в обоих направлениях, что позволяет захватить полный
  контекст;
- **Attention (Bahdanau-style)**: на каждом шаге декодирования вычисляются веса внимания по всем выходам энкодера,
  формируется контекстный вектор;
- **Teacher Forcing**: при обучении с вероятностью `teacher_forcing_ratio` модель получает корректные предыдущие токены,
  что ускоряет сходимость;
- **Label Smoothing**: используется сглаживание меток (`label_smoothing=0.0`) для регуляризации;
- **ReduceLROnPlateau**:.learning rate автоматически уменьшается при отсутствии улучшений на валидации;
- **Gradient Clipping**: градиенты обрезаются (`clip`) для предотвращения взрыва;
- **Early Stopping**: обучение останавливается, если val_loss не улучшается в течение заданного числа эпох.
